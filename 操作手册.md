# 跨境快讯日常操作手册

## 你以后每次要怎么弄

如果已经部署到 GitHub Actions，并且 Secrets 配好了，平时不需要每天手动操作。

- 每周一至周五北京时间 10:07 开始，GitHub Actions 会自动运行。为降低 GitHub 整点定时延迟或漏触发影响，程序会在 10:07-10:32 做错峰补偿尝试；成功发送后当天不会重复发送。
- 程序会自动搜索最近 24 小时新闻，按热度和来源优先级发送 TOP10；不足 10 条时按备用策略补位，仍不足则不硬凑。
- 超过 7 天的旧闻会被硬性过滤。
- 已成功发送过的 URL 和事件指纹会进入历史库，后续 7 天内不再重复发送。
- AMZ123 跨境早报已作为补充线索来源接入；优先保留亚马逊和北美相关新闻，并按 AMZ123 页面顺序发布。AMZ123 的 CPSC/召回类内容允许进入快讯，但来源仍显示 AMZ123；官方 CPSC 来源抓到时优先级更高。
- 快讯标题下方会自动显示今日美元兑人民币中间价，来源为中国人民银行“人民币汇率中间价公告”栏目；不需要额外配置 API 密钥。
- 程序会自动去重、评分、生成快讯并推送到钉钉群。
- 已推送过的 URL 和事件指纹会保存在 `data/history.sqlite`，7 天内不重复推送，30 天后自动清理。

## 什么时候需要你手动操作

需要临时测试或临时补发时：

1. 打开 GitHub 仓库。
2. 点击顶部 `Actions`。
3. 选择 `Cross-border DingTalk Daily`。
4. 点击 `Run workflow`。
5. 参数 `dry_run`：
   - `false`：真实发送到钉钉群，手动运行默认就是这个。
   - `true`：只生成报告，不发送钉钉，适合预览。
6. 点绿色按钮运行。

## 第一次部署需要做什么

第一次需要完成以下一次性设置：

1. 把本项目上传到 GitHub 仓库。
2. 在 GitHub 仓库里打开 `Settings -> Secrets and variables -> Actions`。
3. 添加下面这些 Repository secrets：
   - `DINGTALK_WEBHOOK`
   - `DINGTALK_SECRET`
   - `AI_API_KEY`
   - `AI_BASE_URL`
   - `AI_MODEL`
4. 回到 `Actions` 页面，先手动运行一次，`dry_run` 选 `true`。
5. 确认报告内容没问题后，再手动运行一次，`dry_run` 选 `false`。
6. 后面就等每个工作日北京时间 10:07 开始自动推送；如果 GitHub 排队，可能会在 10:07-10:32 的补偿窗口内完成。

## 更稳妥的备用触发

GitHub Actions 自带 `schedule` 偶尔会延迟或漏触发。更稳妥的做法是保留 GitHub Actions 作为发送程序，再增加一个外部定时器，在工作日北京时间 10:10 或 10:20 调用 GitHub 的 `workflow_dispatch`。

备用触发只需要 GitHub Token，不需要钉钉 Webhook 和加签密钥。钉钉密钥仍然只保存在 GitHub Secrets。

调用地址：

```text
POST https://api.github.com/repos/liyu78763-cmyk/-/actions/workflows/daily-crossborder-news.yml/dispatches
```

请求体：

```json
{"ref":"main","inputs":{"dry_run":"false"}}
```

GitHub Token 权限建议只给本仓库 `Actions: read and write`。

## 本地手动测试

只生成快讯，不发送钉钉：

```powershell
.\.venv\Scripts\python.exe -m crossborder_daily --dry-run --output data\latest_report.md
```

使用测试数据，不访问真实新闻源、不发送钉钉：

```powershell
.\.venv\Scripts\python.exe -m crossborder_daily --dry-run --fixture tests\fixtures\sample_news.json --output data\latest_report.md --no-ai --no-exchange-rate
```

真实发送前，请先临时设置环境变量，不要写入项目文件：

```powershell
$env:DINGTALK_WEBHOOK="你的钉钉Webhook"
$env:DINGTALK_SECRET="你的SEC开头加签密钥"
.\.venv\Scripts\python.exe -m crossborder_daily --output data\latest_report.md
```

## 密钥注意事项

- 不要把 Webhook、加签密钥、AI Key 写到 README、代码、`.env.example` 或 GitHub 仓库文件里。
- GitHub 上只放到 `Settings -> Secrets and variables -> Actions`。
- 如果密钥曾经发到聊天、邮件或公开文档里，建议重新生成钉钉机器人或更换加签密钥。

## 结果怎么看

GitHub Actions 每次运行后：

- `Actions` 页面能看到成功或失败。
- 运行详情里会上传 `crossborder-daily-report-运行编号` artifact。
- artifact 里包含生成的 `data/latest_report.md`。
- 如果不是 dry-run，钉钉群会收到快讯。

## 常见情况

- 群里没收到：先看 GitHub Actions 里有没有今天的运行记录；如果没有，说明 GitHub 定时器没触发，可以手动 Run workflow 补发，或配置备用触发。
- Actions 失败：打开失败步骤日志，看是安装、测试、采集新闻还是钉钉发送失败。
- 新闻很少：这是正常情况，程序不会为了凑数发送低价值或不可核验信息。
- AI Key 没配：程序仍可运行，会使用确定性模板生成快讯，但少一层 AI 润色。
- 钉钉提示签名错误：检查 `DINGTALK_SECRET` 是否是当前机器人的 `SEC...` 加签密钥。
